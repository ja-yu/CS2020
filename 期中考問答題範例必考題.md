```
HTTP
1.HTTP英文全名為HyperText Transfer Protocol
2.HTTP運作在Application layer
3.HTTP是一個客戶端和伺服器端之間請求和應答的標準
4.通常使用TCP協定
5.相關工具:網頁瀏覽器、網路爬蟲

HTTPS
1.HTTPS英文全名為HyperText Transfer Protocol Secure
2.HTTPS運作在Application layer
3.經由HTTP進行通訊，但利用SSL/TLS來加密封包。
4.是提供對網站伺服器的身分認證，保護交換資料的隱私與完整性。
5.對竊聽和中間人攻擊提供合理的防護。

FTP

4.FTP 有兩種傳輸模式：主動和被動。
5.所有傳輸通過明文進行的，在網路上執行封包擷取的人都可以讀取用戶名稱、密碼、命令內容和資料。
```

```
HTTP
超文本傳輸協定（HyperText Transfer Protocol)
應用層（Application layer）位於OSI模型的第七層
HTTP是一個客戶端（用戶）和伺服器端（網站）之間請求和應答的標準

HTTP客戶端發起一個請求，建立一個到伺服器指定埠（初始為80埠）的TCP連線。HTTP伺服器則在那個埠監聽客戶端的請求

超文本傳輸安全協定（Hypertext Transfer Protocol Secure, 常稱為HTTP over TLS、HTTP over SSL或HTTP Secure）
HTTPS經由HTTP進行通訊，但利用SSL/TLS來加密封包
是提供對網站伺服器的身分認證，保護交換資料的隱私與完整性

檔案傳輸協定（File Transfer Protocol）是一個用於在電腦網路上在客戶端和伺服器之間進行檔案傳輸的應用層協定

SSH檔案傳輸協定（SSH File Transfer Protocol，也稱Secret File Transfer Protocol，中文：安全檔案傳送協定，英文：Secure FTP或字母縮寫：SFTP）是一數據流（Data stream）連線，提供檔案存取、傳輸和管理功能的網路傳輸協定
可以為傳輸文件提供一種安全的加密方法

FTPS是一種對常用的檔案傳輸協定（FTP）添加傳輸層安全（TLS）和安全通訊協定（SSL）加密協定支援的擴充協定
加密版的FTP

Telnet是一種應用層協定，使用於網際網路及區域網中，使用虛擬終端機的形式，提供雙向、以文字字串為主的命令列介面互動功能
常用於伺服器的遠端控制，可供使用者在本地主機執行遠端主機上的工作

Secure Shell（安全外殼協定，簡稱SSH）是一種加密的網路傳輸協定，可在不安全的網路中為網路服務提供安全的傳輸環境

傳統Telnet會話所傳輸的資料並未加密，帳號和密碼等敏感資料容易會被竊聽，因此很多伺服器都會封鎖Telnet服務，改用更安全的SSH

簡單郵件傳輸協定（Simple Mail Transfer Protocol，縮寫：SMTP）是一個在網際網路上傳輸電子郵件的標準。
SMTP是一個相對簡單的基於文字的協定

郵局協議（英語：Post Office Protocol，縮寫：POP）此協定主要用於支援使用客戶端遠端管理在伺服器上的電子郵件。最新版本為POP3，全名「Post Office Protocol - Version 3」，而提供了SSL加密的POP3協定被稱為POP3S

網際網路資訊存取協定（英語：Internet Message Access Protocol，縮寫：IMAP；以前稱作互動郵件存取協定）是一個應用層協定，用來從本地郵件客戶端（如Microsoft Outlook、Outlook Express、Foxmail、Mozilla Thunderbird）存取遠端伺服器上的郵件

SMTP是一個「推播「協定，它不允許根據需要從遠端伺服器上「拉取」訊息。要拉取訊息到客戶端，郵件客戶端必須使用POP3或IMAP。
```
