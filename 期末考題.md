# 重要名詞解釋
```
Von Neumann architecture
1.是一種將程式指令記憶體和資料記憶體合併在一起的電腦設計概念架構。
2.電腦被分成了控制單元、算術邏輯單元(ALU)、記憶體、「輸入設備」（Input）與「輸出設備」（Output）。
3.將儲存裝置與中央處理器(CPU)分開
4.本架構設計出的計算機又稱儲存程式電腦。

虛擬記憶體
1.是作業系統具備的記憶體管理技術
2.將資料暫時儲存在儲存媒體中，直到它能被置換進入RAM為止
3.可以應付應用程式的記憶體需求


POST
1.中文為開機自我檢測
2.是BIOS的一個功能
3.會測試電腦的全部元件是否能正常運作
4.檢測到錯誤時，會在螢幕上顯示錯誤資訊或以警報聲響次數指出檢測到的故障

machine cycle(instruction cycle)
1.中文為機器週期(指令週期)
2.是指CPU在執行每個指令經過的步驟
3.步驟為擷取(fetch)、解碼(decode)、執行(execute)、儲存(store)
4.擷取、解碼指令由控制單元負責
5.執行、儲存指令由ALU負責
```
# 簡答題
```
作業系統的功能
1.電腦或裝置的開機和關機
2.管理程序
3.管理記憶體
4.協調工作
5.設定裝置
6.建立Internet連線
7.監控效能
8.提供管理檔案功能
9.更新作業系統軟體
10.監控安全性
11.控制網路存取

Memory hierarchy
在電腦架構下儲存系統階層的排列順序。
每一層於下一層相比都擁有較高的速度和較低延遲性，以及較小的容量、較高的價格
第一層:暫存器
第二層:快取(L1～L3:SRAM、L4)
第三層:主記憶體（DRAM）
第四層:硬碟
```

# 負數表達法

## 題目:-39

## (1)signed-magniture
```
step1:取數字的正數部分

-39 ==> 39

step2:將正數部分轉換成7個位元的二進位值

39 ==> 0100111

step3:在最前面加上符號位元，正數為0，負數為1

0100111 ==> 10100111
```
## (2)1's complement
```
step1:取數字的正數部分

-39 ==> 39

step2:將正數部分轉換成8個位元的二進位值

39 ==> 00100111

step3:將每個位元反相

00100111 ==> 11011000
```
## (3)2's complement
```
step1:取數字的正數部分

-39 ==> 39

step2:將正數部分轉換成8個位元的二進位值

39 ==> 00100111

step3:將每個位元反相

00100111 ==> 11011000

step4:將反相後二進位值加1

11011000 + 1 ==> 11011001
```
## 題目:10010010

## (1)signed-magniture
```
step1: 判斷正負
       數字最前面位元數判斷，正數為0，負數為1

[1]0010010 ==> 負

step2: 計算大小(magnitude)
       抽出後面7個位元計算成十進位

0010010 = 16 + 2 = 18

step3: 組合正負號、大小

正負號、大小 ==> -18
```
## (2)1's complement
```
step1: 判斷正負
       數字最前面位元數判斷，正數為0，負數為1

[1]0010010 ==> 負

step2: 計算大小(magnitude)
step2-1: 先執行反相(inverse)

10010010 ==> 01101101

step2-2: 計算成十進位

01101101 = 64 + 32 + 8 + 4 + 1 = 109

step3:組合正負號、大小
正負號、大小 ==> -109
```
## (3)2's complement
```
step1: 判斷正負
       數字最前面位元數判斷，正數為0，負數為1

[1]0010010 ==> 負

step2: 計算大小(magnitude)
step2-1: 先減1

10010010 - 1 = 10010001

step2-2: 執行反相(inverse)

10010001 ==> 01101110

step2-3: 計算成十進位

01101110 = 64 + 32 + 8 + 4 + 2 = 110

step3:組合正負號、大小
正負號、大小 ==> -110
```
